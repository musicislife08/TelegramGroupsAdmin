Refactor the OTEL configuration block in Program.cs (lines 143-174) and the endpoint mapping block (lines 340-345). The change is a conditional logic expansion, not an architecture change.

**Step 1 — Read ENABLE_METRICS env var (after existing otlpEndpoint read on line 145):**

Add a `metricsEnabled` bool that OR's the two env vars:
```csharp
var metricsEnabled = !string.IsNullOrEmpty(otlpEndpoint)
                  || !string.IsNullOrEmpty(builder.Configuration["ENABLE_METRICS"]);
```

**Step 2 — Split the single if-block into two independent blocks:**

Block 1 — Metrics pipeline (activates on `metricsEnabled`):
```csharp
if (metricsEnabled)
{
    builder.Services.AddOpenTelemetry()
        .ConfigureResource(resource => resource
            .AddService(serviceName, serviceVersion: serviceVersion)
            .AddAttributes(new Dictionary<string, object>
            {
                ["deployment.environment"] = environment
            }))
        .WithMetrics(metrics =>
        {
            metrics.AddAspNetCoreInstrumentation()
                   .AddHttpClientInstrumentation()
                   .AddRuntimeInstrumentation()
                   .AddMeter("Npgsql")
                   .AddMeter("TelegramGroupsAdmin.*")
                   .AddPrometheusExporter();

            if (!string.IsNullOrEmpty(otlpEndpoint))
                metrics.AddOtlpExporter();
        });
}
```

Block 2 — Tracing pipeline (activates on `otlpEndpoint` only):
```csharp
if (!string.IsNullOrEmpty(otlpEndpoint))
{
    builder.Services.AddOpenTelemetry()
        .ConfigureResource(resource => resource
            .AddService(serviceName, serviceVersion: serviceVersion)
            .AddAttributes(new Dictionary<string, object>
            {
                ["deployment.environment"] = environment
            }))
        .WithTracing(tracing => tracing
            .AddAspNetCoreInstrumentation()
            .AddHttpClientInstrumentation()
            .AddSource("Npgsql")
            .AddSource("TelegramGroupsAdmin.*")
            .AddOtlpExporter());
}
```

Note: `ConfigureResource()` is called in both blocks with identical values — this is safe (harmless duplication) and guarantees correct resource attributes regardless of which combination of env vars is active. Per research, multiple calls to `AddOpenTelemetry()` on the same `IServiceCollection` are safe in OTEL .NET Extensions.Hosting (they share the same underlying builder).

**Step 3 — Update the endpoint mapping block (lines 340-345):**

Replace the `otlpEndpoint` condition with `metricsEnabled` and add activation source to log:
```csharp
if (metricsEnabled)
{

        ? "OTEL_EXPORTER_OTLP_ENDPOINT"
        : "ENABLE_METRICS";
    app.Logger.LogInformation(
        "Prometheus metrics endpoint mapped to /metrics (via {ActivatedBy})", activatedBy);
}
```
