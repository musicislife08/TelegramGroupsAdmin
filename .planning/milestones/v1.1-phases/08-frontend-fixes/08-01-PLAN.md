<objective>
Fix analytics overview card growth percentages so they always show week-over-week comparison regardless of the selected time range filter, add a dedicated DailyAverageGrowthPercent, and hide chips on All Time view.

Purpose: The 7-day view currently shows no growth percentages because the growth calculation requires 14 days of data in the date range. Growth is always week-over-week and should be calculated from `endDate` backward, independent of the selected `startDate`. The Daily Average card incorrectly reuses `MessageGrowthPercent`.

Output: Corrected growth calculation in MessageStatsService, updated WeekOverWeekGrowth model, updated MessageTrends.razor, and unit tests covering the fix.
</objective>

Bug root cause in MessageStatsService.cs line 320:
```csharp
var hasPreviousPeriod = daysDiff >= AnalyticsConstants.MinDaysForWeekOverWeekGrowth;
```
`daysDiff` is derived from `(endDate - startDate)`. For 7-day view, daysDiff=7, which is < 14, so growth is never computed. The fix: growth should always be computed from the last 14 days relative to `endDate`, regardless of the `startDate` filter.
