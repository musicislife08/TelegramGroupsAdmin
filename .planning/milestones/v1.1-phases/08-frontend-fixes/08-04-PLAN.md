From MainLayout.razor (timezone detection pattern):
```csharp
protected override async Task OnAfterRenderAsync(bool firstRender)
{
    if (firstRender)
    {
        try
        {
            var tzId = await JsRuntime.InvokeAsync<string>("getUserTimeZone");
            _userTimeZone = TimeZoneInfo.FindSystemTimeZoneById(tzId);
        }
        catch
        {
            _userTimeZone = TimeZoneInfo.Utc;
        }
        StateHasChanged();
    }
}
```
The key safety: `OnAfterRenderAsync` only fires AFTER SignalR connects (post-prerender), so `JsRuntime.InvokeAsync` never runs during the prerender SSR pass where JS interop is unavailable.

**Test: `ColdPageLoad_NoJSExceptionInConsole`**

1. Set up a console error collector BEFORE navigating:
   ```csharp
   var consoleErrors = new List<string>();
   Page.Console += (_, msg) =>
   {
       if (msg.Type == "error")
           consoleErrors.Add(msg.Text);
   };
   ```

2. Log in using `await LoginAsOwnerAsync()` (backend cookie injection — no login UI)

3. Navigate to the home page (`await NavigateToAsync("/")`) — this is a cold page load that triggers the Blazor prerender → circuit connect lifecycle. The home page uses `LocalTimestamp` components (e.g., in the dashboard).

4. Wait for the page to settle: `await Page.WaitForLoadStateAsync(LoadState.NetworkIdle)`

5. Also wait briefly for any delayed errors: `await Task.Delay(1000)` — some JS interop errors fire slightly after network idle when the circuit connects.

6. Assert no console errors contain JSException or JavaScript interop keywords:
   ```csharp
   var jsInteropErrors = consoleErrors
       .Where(e => e.Contains("JSException", StringComparison.OrdinalIgnoreCase)
                 || e.Contains("JavaScript interop calls cannot be issued", StringComparison.OrdinalIgnoreCase)
                 || e.Contains("InvalidOperationException", StringComparison.OrdinalIgnoreCase))
       .ToList();

   Assert.That(jsInteropErrors, Is.Empty,
       $"Found {jsInteropErrors.Count} JS interop error(s) during cold page load: {string.Join("; ", jsInteropErrors)}");
   ```
