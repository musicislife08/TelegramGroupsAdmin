<decisions>
## Implementation Decisions

### Critical fixes (must do)
- **Vacuous test assertions in MessageStatsServiceTests**: Tests 3 and 5 use `if (growth != null) { Assert... }` pattern that passes silently if growth is null. Replace with unconditional assertions — Test 3 should assert growth is null (only 1 week of data), Test 5 should assert the specific zero-division-safe state.
- **Empty catch in MainLayout.razor timezone detection** (line ~131): Add `ILogger<MainLayout>` injection and log at Warning level with the failed timezone ID. `catch (Exception ex) { Logger.LogWarning(ex, "Failed to detect timezone..."); _userTimeZone = TimeZoneInfo.Utc; }`

### Included suggestions
- **UTC timestamp placeholder (#3)**: Update `LocalTimestamp.razor` to render `---` (or non-breaking space) when `UserTimeZone` is null, instead of showing UTC times that then shift. Timestamps appear once in the correct timezone.
- **Private CascadingParameter (#4)**: Change `[CascadingParameter] public TimeZoneInfo? UserTimeZone` to `private` in all 4 components (MessageTrends, PerformanceMetrics, StopWordRecommendations, LocalTimestamp). Matches project convention (22+ components use `private`).
- **Fire-and-forget CancellationToken (#5)**: Change `ct: cancellationToken` to `cancellationToken: CancellationToken.None` at both fire-and-forget call sites in `ChatHealthRefreshOrchestrator` (lines ~207, ~259). Also fix the parameter name from `ct` to `cancellationToken` per project convention.
- **Remove Task.Delay from E2E tests (#7)**: Replace `Task.Delay(1000)` in `TimezonePreRenderTests.cs` with proper Playwright Expect auto-retry or WaitForLoadStateAsync. Task.Delay should NEVER be used in E2E tests.
- **Null WeekOverWeekGrowth test (#8)**: Add one bUnit test to `MessageTrendsOverviewCardTests` with `growth: null` to verify no NullReferenceException.

### Skipped / deferred
- **#6 (PerformanceMetrics timezone data reload)**: Deferred to #406. Narrow edge case (direct-link on first visit), self-corrects on interaction. Needs investigation into whether timezone actually affects SQL bucketing.

### Testing
- Unit test fixes are the deliverable (fixing existing tests, adding null growth test)
- No new production test classes needed — this is fixing test quality
- E2E test improvement is fixing an existing test

### Claude's Discretion
- Exact placeholder rendering in LocalTimestamp when timezone is null (dash, empty, skeleton)
