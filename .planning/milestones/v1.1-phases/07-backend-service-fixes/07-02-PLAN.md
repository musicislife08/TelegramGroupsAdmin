<objective>
Wire up the health orchestrator's unused CheckHealthAsync and MarkInactiveAsync methods (#333).

Purpose: The `ChatHealthRefreshOrchestrator` performs its own inline health checks instead of delegating to `BotChatService.CheckHealthAsync`, and never calls `ManagedChatsRepository.MarkInactiveAsync` when a chat becomes unreachable. This means missed `MyChatMember` events (e.g., bot kicked during a network blip) leave stale active chats in the database forever. Wiring these methods creates a safety net that detects and marks unreachable chats as inactive after 3 consecutive failures.

Output: Orchestrator delegates to CheckHealthAsync, tracks failures in-memory, marks chats inactive after 3 strikes, with full unit test coverage.

2. **Wire CheckHealthAsync into the health check flow:**
   In `RefreshHealthForChatAsync`, BEFORE the existing `PerformHealthCheckAsync` call, add a preliminary reachability check using `chatService.CheckHealthAsync(chat, cancellationToken)`:

   ```csharp
   // Quick reachability check via BotChatService (delegates to Telegram API)
   var isReachable = await chatService.CheckHealthAsync(chat, cancellationToken);

   if (!isReachable)
   {
       // Track consecutive failures
       var failureCount = failureTracker.IncrementFailureCount(chat.Id);
       logger.LogDebug("Health check failed for {Chat}, consecutive failures: {Count}",
           chat.ToLogDebug(), failureCount);

       if (failureCount >= 3)
       {
           logger.LogWarning("Chat {Chat} unreachable after {Count} consecutive failures, marking inactive",
               chat.ToLogDebug(), failureCount);
           await managedChatsRepository.MarkInactiveAsync(chat, cancellationToken);
           failureTracker.ResetFailureCount(chat.Id); // Reset after marking inactive
       }

       // Set error health status in cache
       var errorHealth = new ChatHealthStatus
       {
           Chat = chat,
           IsReachable = false,
           Status = ChatHealthStatusType.Error
       };
       errorHealth.Warnings.Add("Chat unreachable - bot may have been removed");
       healthCache.SetHealth(chat.Id, errorHealth);
       return;
   }

   // Reachable - reset failure counter
   failureTracker.ResetFailureCount(chat.Id);
   ```

   Then continue with the existing `PerformHealthCheckAsync` call for the detailed health check (permissions, admin status, etc.) as before.

   **IMPORTANT:** The goal is to add the CheckHealthAsync + MarkInactiveAsync wiring WITHOUT removing the existing detailed health check logic. The `CheckHealthAsync` is a quick "can we reach this chat at all?" gate. If reachable, the existing `PerformHealthCheckAsync` still runs for the detailed permission/admin checks.
