## Integration Issues Found

### High: Compose env var name mismatch (CLAM-01 through CLAM-04)

**Problem:** `examples/compose.development.yml` and `examples/compose.production.yml` set `CLAMAV__HOST`/`CLAMAV__PORT` (double-underscore — ASP.NET Core `IConfiguration` binding). Phase 9 reads `CLAMAV_HOST`/`CLAMAV_PORT` (single-underscore — raw `Environment.GetEnvironmentVariable`).

**Impact:** The env var override feature is unreachable from the canonical compose examples. Operators would need to add the single-underscore variants as additional env vars.

**Code is correct** — this is purely a deployment documentation gap.

### Low: `--bootstrap` undocumented in compose/CLAUDE.md (BOOT-07)

The `--bootstrap` flag works correctly but neither compose files nor CLAUDE.md show the K8s init container usage pattern.

### Low: Production compose missing `ENABLE_METRICS` (STAT-01)

`compose.production.yml` includes neither `ENABLE_METRICS` nor `OTEL_EXPORTER_OTLP_ENDPOINT`. The development compose is fine (sets `OTEL_EXPORTER_OTLP_ENDPOINT`).

### Info: Stale REQUIREMENTS.md text (STAT-01/02/03)

REQUIREMENTS.md references `SEQ_URL` as the prior gate for `/metrics` but actual pre-phase gate was `OTEL_EXPORTER_OTLP_ENDPOINT`. Implementation is correct; only the requirements doc text is stale.
